# Future Plans

The architecture is expected to evolve incrementally.

## Short Term

* Docker platform
* Core networking
* Nextcloud
* Immich
* Homepage
* Uptime Kuma

## Medium Term

* Paperless-ngx
* Vaultwarden
* Mealie
* Syncthing
* Monitoring improvements

## Long Term

* Mini PC migration
* Multi-disk storage
* Infrastructure automation
* Centralized logging
* Metrics platform
* CI/CD for infrastructure
* Automated disaster recovery testing

## Architectural Rule

Growth should happen by adding independent services.

Existing components should rarely require redesign.

The architecture should become more modular over time—not more complex.
